# Platform Setup (Android / iOS)

This repo contains `lib/`, `test/`, `pubspec.yaml`, and docs only — the
`android/`, `ios/`, and other platform folders are **not included**, since
they're normally generated by the `flutter create` tool, which wasn't
available in the environment this was built in.

## 1. Generate the platform folders

```bash
flutter create . --project-name ebook_library_frontend --org com.sagarfab
```

Run this **in this project's root** (where `pubspec.yaml` already lives) —
it fills in the missing `android/`, `ios/`, `web/`, etc. folders around the
existing `lib/`/`test/` without touching them.

## 2. Required permissions once the folders exist

### Android — `android/app/src/main/AndroidManifest.xml`

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

`INTERNET` is usually included by default by `flutter create`, but double
check it's there — without it, every API call fails silently with a
connection error.

**If your backend runs on plain `http://` during development** (which
`bin/rails server` does by default), Android 9+ blocks cleartext traffic
unless you allow it. Either add this to the `<application>` tag:

```xml
<application
    android:usesCleartextTraffic="true"
    ...>
```

or scope it to just your dev host via a network security config (preferred
for anything beyond local testing) — see
[Android's cleartext traffic docs](https://developer.android.com/privacy-and-security/security-config) for the network-security-config.xml approach.

### iOS — `ios/Runner/Info.plist`

Same issue, iOS side — `http://` to a local dev server needs an App
Transport Security exception:

```xml
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <true/>
</dict>
```

Scope this to development builds only; a production build should talk to a
real `https://` backend and not need this exception at all.

Also add (used by `file_picker` for selecting cover images from Photos):

```xml
<key>NSPhotoLibraryUsageDescription</key>
<string>Used to select a cover image for your ebook.</string>
```

### Both platforms — file storage

`open_filex` and `path_provider` (used for downloads) work out of the box
on both platforms with `flutter create`'s defaults — no extra permission
entries needed beyond the above for the app's own sandboxed documents
directory.

## 3. Minimum SDK versions

`syncfusion_flutter_pdfviewer` requires:
- Android: `minSdkVersion 21` or higher (`android/app/build.gradle`)
- iOS: deployment target `12.0` or higher (`ios/Podfile`)

`flutter create`'s current defaults already satisfy both, but worth
confirming if you're merging into an older existing project.
